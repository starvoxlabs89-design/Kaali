Three surfaces. One stack.

Kaali is open source — MIT — and made of three pieces you can adopt separately. Together they cover the surface every AI app leaves exposed.

◈

Scanner (CLI)

npx @kaali/cli scan against a URL, a repo, or an MCP server. Finds missing security headers, hardcoded API keys, exposed Aadhaar / PAN, invisible-Unicode prompt injection, MCP tool-poisoning.

◆

Guard (runtime)

npm i @kaali/guard — inline Express/Next middleware. Blocks prompt injection, decodes hidden Unicode, redacts Indian PII in flight, firewalls MCP tool calls. Two microseconds per request.

◇

Cloud (24/7)

Hosted dashboard. Continuous monitoring of your registered targets, historical trend charts, DPDP / OWASP-LLM evidence reports, Slack alerts. Free while in beta.

What Kaali watches for.

The threats no observability tool sees. Every one of these is a real attack pattern in 2026 — not a theory.

OWASP LLM01

Prompt injection

Direct overrides ("ignore previous…"), role hijacks, delimiter breaks, data-exfil traps.

GlassWorm class

Invisible-Unicode

Instructions smuggled to AIs in U+E00xx tag chars, zero-width, and bidi overrides — invisible to humans. Kaali decodes them.

Agentic

MCP tool poisoning

Malicious instructions hidden in Model Context Protocol tool descriptions. Kaali audits deployed MCP servers.

DPDP §11

Aadhaar / PAN leaks

Personal data spilling through code, logs, or LLM responses. Verhoeff-validated (no false positives).

Web hygiene

Missing headers & secrets

HSTS, CSP, XFO, XCTO, Referrer-Policy — and OpenAI / AWS / Anthropic keys hardcoded in source.

Agent runtime

Indirect injection

Poisoned web pages and tool results hijacking your agent (Greshake et al., 2023). Live probes.
